```verilog
module fsm_seq_detector(input clk, rst, in, output wire detected);
    reg [1:0] state;
    
    always @(posedge clk or posedge rst) begin
        if (rst) 
            state <= 2'd0; // Reset to S0 on reset signal
        else 
            case (state)
                2'd0: state <= in ? 2'd1 : 2'd0; // If input is 1, go to S1, stay at S0 otherwise
                2'd1: state <= !in ? 2'd2 : 2'd1; // If input is 0, go to S2, stay at S1 otherwise
                2'd2: state <= in ? 2'd3 : 2'd0; // If input is 1, go to S3, reset to S0 otherwise
                default: state <= !in ? 2'd3 : 2'd2; // For state S3, if input is 0, go back to S2, stay at S3 otherwise
            endcase
    end
    
    assign detected = (state == 2'd3) & in; // Output 'detected' fires when the state is S3 and input is 1
endmodule
```
This corrected code should now produce the expected output. The FSM transitions as follows:
- When `in` is 0, it stays at S0 (2'd0) or moves to S1 (2'd1).
- When `in` is 1 and state is S0 (2'd0), it goes to S1 (2'd1).
- When `in` is 1 and state is S1 (2'd1), it stays at S2 (2'd2) or moves to S3 (2'd3) if the input is 1.
- When `in` is 0, it stays at S2 (2'd2) or goes back to S1 (2'd1) in state S3 (2'd3).
The output 'detected' will be 1 when the state is S3 and input is 1.
